# AWS Setup Guide: Emotion-Based Music Recommendation System

Follow these steps to deploy the application on AWS.

## 1. Amazon S3 (Music Storage)
1. Go to the **S3 Console**.
2. Create a bucket named `emotion-music-storage` (or your preferred name).
3. Upload your music files into folders (e.g., `happy/`, `sad/`, `angry/`, `calm/`).
4. Ensure the bucket is private (default). Access will be via pre-signed URLs generated by Lambda.

## 2. Amazon DynamoDB (Database)
1. Go to the **DynamoDB Console**.
2. Create a table named `MusicCatalog`:
   - **Partition Key**: `emotion` (String)
   - **Sort Key**: `songId` (String)
3. Create a table named `Users`:
   - **Partition Key**: `userId` (String)
4. Use the `scripts/seed_dynamodb.py` script provided in the codebase to populate the `MusicCatalog` table.

## 3. IAM Role (Security)
1. Go to the **IAM Console** > **Roles**.
2. Create a role for **Lambda** that has the following policies:
   - `AmazonS3ReadOnlyAccess`
   - `AmazonDynamoDBFullAccess`
   - `ComprehendReadOnly`
   - `AmazonRekognitionReadOnlyAccess`
   - `CloudWatchLogsFullAccess` (for debugging)

## 4. AWS Lambda (Backend Logic)
1. Create a function named `EmotionMusicBackend` using **Python 3.9+**.
2. Attach the IAM role created in Step 3.
3. Copy the code from `backend/lambda_handler.py` into the Lambda editor.
4. Set the following **Environment Variables** in the "Configuration" tab:
   - `MUSIC_TABLE`: MusicCatalog
   - `S3_BUCKET`: emotion-music-storage
5. Increase the **Timeout** to `10 seconds` (AI services can take a moment).

## 5. Amazon API Gateway (API)
1. Go to **API Gateway Console** > **Create API** > **HTTP API**.
2. Add an integration with your Lambda function `EmotionMusicBackend`.
3. Create a `POST` route named `/analyze`.
4. **Enable CORS**:
   - Access-Control-Allow-Origin: `*`
   - Access-Control-Allow-Methods: `POST, OPTIONS`
   - Access-Control-Allow-Headers: `Content-Type`
5. Deploy the API and note the **Invoke URL**.

## 6. Frontend Deployment
1. Open `frontend/src/App.jsx` and replace `YOUR_API_GATEWAY_URL` with your API Invoke URL (e.g., `https://xyz.execute-api.us-east-1.amazonaws.com/analyze`).
2. Run locally: `npm install && npm run dev`.
3. (Optional) For production, build the app (`npm run build`) and upload the `dist/` folder to an S3 bucket configured for **Static Website Hosting**.
4. (Optional) Point **Amazon CloudFront** to your S3 bucket for HTTPS and global delivery.

## 7. Testing
- **Text**: Type "I am so happy today!" and click "Analyze".
- **Image**: Upload a clear selfie showing an expression.
- **Result**: You should see the detected emotion and a list of songs retrieved from DynamoDB.
